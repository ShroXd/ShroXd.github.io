站会常用表达
2021-04-14|英语
Word count: 414|Reading time: 1 min

简述

站会时所可能用到的表达方式主要有两个方面：

  1. 与其他人对接工作
  2. 主持站会
    其中，以情况 1 的优先级更高，所以本文将优先总结第一种情况。

对接工作

在对接工作时，我们需要准确的表达系统的设计，api 的字段需求以及遇到的一些问题。同时，同意/不同意对方的观点，补充说明，请求对方重新解释等，都是常常需要用到的句式。下面就这些问题一一总结。

如何表达设计

  1. I’m going to spend a few minutes talking about the problem with this card.

  2. I’m going to discuss the design of the system.

  3. We want to discuss some of the problems we have encountered at this stage.

如何表达工作进度

  1. I haven’t started the work yet.

    • I have some difficulty in doing something.
    • There are some problem with noun.
    • 我们需要对接一下 api 请求的字段

  2. I’m already doing the work.

    • So if all goes well, i can finish it on Friday.

  3. I have already finished the work, so we can move the card to the xxx phase.

  4. I have fixed the error, so our next work is to improve this component. Complete the style and logic of the component.

如何确认需求

  1. So my question is, should I add this field to Users enumeration?

  2. Which UI design should I follow?

  3. I notice that this field is not in the document, but I need to add it when I send the request to the back-end, otherwise I will get an error.

如何表达工程上的取舍

  1. The advantage of this is that we can selectively export components.

  2. This work has a high priority.

  3. The priority of this work is not high.

如何表达不同意对方观点

  1. I’m not sure if it’s a good design/decision.
  2. I think we can go a step further and abstract out the entire component.
